# Post-Training Experiment Scope

Analogous to the pre-training experiment loop (milestone 7), this defines the cheap validation layer before committing to the full post-training run on the 3B checkpoint.

## Core Constraint

**SFT ablations scale down cleanly. RL does not.**

For pre-training, a 21M model validates recipe decisions (learning rate, batch size, data mix) before committing to 3B. SFT works the same way — relative differences between ablation arms hold at small scale even if absolute numbers don't. RL breaks this: a model too small to generate coherent rollouts produces meaningless reward signals. The first RL run IS the experiment.

This splits the design into three tiers:

| Tier | Base model | Purpose | Cost |
|---|---|---|---|
| SFT ablations | 330M experiment checkpoint | Validate SFT recipe (mix, loss, data quantity) | ~$1–4/run |
| RL baseline + sweeps | 3B post-SFT checkpoint | Validate RL setup; sweep G and curriculum | ~$16–100/run |
| Thinking decision | 3B post-RL checkpoint | Binary: does thinking help >10%? | ~$8–24 |

---

## Tier 1: SFT Ablations (330M)

Use the existing 330M pre-trained experiment checkpoint as the base. Run all ablations to completion (3 epochs), evaluate on the held-out trace pass@1 + IFEval. Pick the winning arm, then apply it to the 3B.

### Ablations

All arms share: 20K traces, 3 epochs, TRL SFTTrainer, 6144 ctx, AdamW lr=1e-5, cosine warmup 3%.

| ID | Variable | Arm A (baseline) | Arm B | Metric |
|---|---|---|---|---|
| S1 | Instruction mix | 70% traces / 30% instruction | 100% traces | pass@1 + IFEval |
| S2 | Loss formulation | Response-only | Instruction modelling (full seq) | pass@1 |
| S3 | Trace quantity | 20K traces | 5K traces | pass@1 (data efficiency) |
| S4 | Trace quantity | 20K traces | 50K traces | pass@1 (diminishing returns) |
| S5 | LoRA vs full | Full fine-tune | LoRA r=64 α=128 | pass@1 + wall time |

Run S1 and S2 first — they answer the most important recipe questions. S3/S4 inform trace generation budget for milestone 3. S5 informs whether to use LoRA on 3B (memory vs quality tradeoff).

### Gating Criterion

S1–S2 must show a clear winner (≥5% pass@1 gap between arms) before proceeding to 3B SFT. If the gap is <5%, flip a coin (they're equivalent) and move on — don't over-tune on the proxy model.

### Catastrophic Forgetting Check

After each SFT arm, run HumanEval (pass@1) on the 330M checkpoint. If HumanEval drops >10% from the pre-SFT baseline, the arm is poisoning code quality. Discard it regardless of trace pass rate.

### Cost

~5 runs × ~1 H100-hour = **~$10–20 total**. Very cheap.

## Tier 2: RL Ablations (3B post-SFT)

The first RL run is the baseline. Experiments are sequential sweeps, not parallel arms — RL is expensive and the signal is noisy enough that running parallel arms wastes budget without improving decision quality.

### Baseline Run

Apply the winning SFT recipe to 3B. Run GRPO with conservative defaults:

| Param | Value | Rationale |
|---|---|---|
| G (group size) | 4 | Conservative; upgrade to 8 if reward stays sparse |
| KL penalty (β) | 0.02 | Center of safe range |
| Steps | 500 | Enough to see convergence trend |
| Curriculum | Easy-first (single-step tasks only first 100 steps) | Prevents zero-gradient start |
| Reward | Binary pass/fail | No shaping initially |

**Success criterion:** average reward rises above 0.2 by step 200. If it doesn't, the reward signal is too sparse — move to Sweep A immediately.

### Sweep A: Curriculum vs. Shaped Rewards (if baseline stalls)

Run only if baseline average reward < 0.2 at step 200. Two rescue arms, run sequentially:

- **A1:** Add reward shaping (+0.1 per successful individual tool call)
- **A2:** Extend curriculum (single-step tasks for first 300 steps instead of 100)

Pick whichever first shows rising reward. This is a triage sweep, not a precision comparison.

### Sweep B: Group Size (after baseline converges)

Once the baseline converges (reward plateaus or pass@1 ≥ 60%), run one more arm:

- **B1:** G=8 vs baseline G=4 (same steps, same budget)

If G=8 shows ≥5% pass@1 improvement, use it for the full post-training run. Otherwise G=4 is fine.

### Monitoring

Every 50 steps, log:
- Mean reward across rollouts
- KL divergence from SFT policy (alert if > 0.1 nats)
- Pass@1 on 20 fixed held-out tasks (quick eval, not full suite)
- Rollout entropy (alert if < 0.5 — policy collapse)

**Abort criterion:** if KL spikes above 0.2 nats at any step, halt and restart with β=0.04.

### Cost

Baseline (500 steps): ~8 H100-hours. Each sweep arm: ~8 H100-hours.
Expected total: **~16–40 H100-hours** ($32–160) depending on how many rescues are needed.

## Tier 3: Thinking Decision (3B post-RL)

Binary experiment. One run, clear pass/fail.

### Setup

- 20K thinking traces generated by the post-RL checkpoint (rejection-filtered)
- 50/50 thinking/non-thinking mix by token count
- 1 epoch, LR=5e-6, max think block 1024 tokens
- Run for 1 epoch, eval on hard task suite (multi-step pipelines)

### Decision Criterion

| Outcome | Action |
|---|---|
| pass@1 on hard tasks improves ≥10% | Ship thinking fusion; include in full post-training run |
| pass@1 improves 5–10% | Run a second epoch; if still <10%, abort |
| pass@1 improves <5% | Abort; 3B lacks capacity for useful self-reasoning at this stage |
| No-think compliance fails | Fix special token handling before any further thinking work |

Cost: **~4–8 H100-hours** ($8–32).

---

## Decision Tree

```
330M SFT ablations (S1–S5)
    → winning recipe
        → 3B SFT (full)
            → RL baseline (500 steps)
                → reward rises? ─── yes → Sweep B (G size)
                                             → full post-training run
                └── no → Sweep A (curriculum / shaping)
                            → winning rescue arm → Sweep B → full run
            → thinking decision (1 run)
                → +10%? include in full run
                └── no → skip
```

---

## What Experiments Cannot Tell You

- **Whether 3B thinking is worth it at all:** only the thinking experiment answers this; no proxy exists.
- **Whether the SFT recipe transfers from 330M to 3B exactly:** ratios and loss formulations should transfer; absolute numbers won't. Don't calibrate pass@1 targets from 330M results.
- **Whether RL converges in 500 steps:** 500 steps is enough to see the trend; if reward is still flat at 500 you have a structural problem. Full convergence likely takes 1000–2000 steps.

## Experiment Readiness Checklist

Before starting Tier 1:
- [ ] 330M experiment checkpoint available (pre-training complete)
- [ ] At least 20K Docker-validated agentic traces ready
- [ ] Special tokens correctly registered in tokenizer (verify no splitting)
- [ ] TRL SFTTrainer packing + per-sample loss normalization confirmed

Before starting Tier 2:
- [ ] Winning SFT recipe chosen from Tier 1
- [ ] 3B checkpoint available (post-SFT)
- [ ] veRL or TRL GRPOTrainer configured on Modal
- [ ] Docker sandbox reward function tested end-to-end (not just mocked)
- [ ] 500+ unique tasks in RL training distribution

Before starting Tier 3:
- [ ] 3B post-RL checkpoint available
- [ ] Think/no-think special tokens in protocol (`<|think|>`, `<|no_think|>` or equivalent)
- [ ] 20K thinking traces generated + filtered
