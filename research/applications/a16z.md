## Email / application text

**Subject:** Open-source 3B LLM optimized for Apple Silicon inference — grant inquiry

We're building Merlin, an open-source 3B language model purpose-built for agentic coding on Apple Silicon. The project matches two of your explicitly stated focus areas:

**"Optimizing models for low-powered machines"** — Merlin targets local inference on any M-series MacBook: int4 quantization, ~1.5GB weights, >500 tok/s on M3 via MLX. The inference stack (MLX kernels, quantization pipeline, weight conversion from PyTorch) is a first-class deliverable alongside the weights.

**"Novel inference tooling"** — we're building a worker-orchestrator architecture where Merlin handles atomic agentic tasks (grep, file read/write, bash execution) while a frontier model handles planning. This two-tier design reduces API costs by 50%+ for power users and enables 100–1000x parallel task execution.

The model is trained from scratch (no proprietary distillation — weights are commercially clean), with a custom tokenizer optimized for code and agent protocol tokens, and post-trained with RL on verifiable bash/filesystem rewards.

What's already shipped:
- 1.19B-token v0 corpus on HuggingFace (tsuberim/merlin-corpus-v0)
- Custom 32K BPE tokenizer with 20 agent protocol special tokens
- 49-task agentic eval harness (47% on 3B-scale baseline)
- Full E2E training loop on H100 via Modal

Total compute budget: ~$5,100 for the full 3B run.

Everything will be open-sourced under MIT: weights, training code, data pipeline, trace dataset, MLX inference stack.
