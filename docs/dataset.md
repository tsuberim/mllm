# Dataset

## v0 Corpus (merlin-corpus-v0)

Pretraining corpus focused on code and technical text. No general web, no Wikipedia.

**1.19B tokens ¬∑ 100% packing efficiency ¬∑ fits in H100 HBM (~2.4 GB)**

HuggingFace: [tsuberim/merlin-corpus-v0](https://huggingface.co/datasets/tsuberim/merlin-corpus-v0)  
Tokenizer: [tsuberim/merlin-tokenizer-v0](https://huggingface.co/tsuberim/merlin-tokenizer-v0)

> v0 does not yet include agentic traces. Tokenizer and corpus will be retrained after trace generation.

## Filters (`01_filter.py`)

Each source has its own filter. Hard rejections ‚ÄĒ no probabilistic sampling.

### Python
- Must parse as valid Python 3 (AST check)
- Must contain at least one `def`, `async def`, or `class` node
- 5‚Äď2000 lines
- No generated files: `DO NOT EDIT`, `AUTO-GENERATED`, swagger/protoc/thrift/avro output, Django migrations, alembic/SQLAlchemy generated code

### Bash
- ‚Č•2 shell command patterns (shebang lines, builtins, common tools like `git`, `docker`, `curl`, `grep`, etc.)
- No generated files

### Markdown
- ‚Č•200 chars
- ‚Č§5% non-ASCII in first 1000 chars (English-only proxy)
- ‚Č§10% of chars inside HTML tags AND ‚Č§20% of lines starting with `<` (drops HTML blobs)
- Must contain at least one backtick (proxy for technical/code content)
- No template placeholders (`[[language]]`, `[[version]]`, etc.)

### Stack Overflow
- ‚Č•300 chars (Q + A combined)
- Must contain a code block (triple-backtick or indented)
- Must match domain keywords **in the question portion only** ‚ÄĒ prevents C# questions that mention "bash" in an answer from passing. Keywords: `python`, `bash`, `shell`, `zsh`, `linux`, `docker`, `kubernetes`, `pip`, `flask`, `django`, `fastapi`, `subprocess`, `asyncio`, `grep`, `curl`, `ssh`, `nginx`, `apt-get`, `brew`, `makefile`, `cmake`, `re.compile`, etc.

### GitHub commits
- ‚Č•128 chars of diff content
- First line (commit message) must not be a trivial template: merge, fix typo, update readme, bump version, wip, minor, cleanup, initial commit, add files, etc.

### GitHub issues
- ‚Č•200 chars
- Not a bot-generated post (`this issue has been automatically...`, `generated by *bot`, etc.)
- Must match domain keywords in first 2000 chars (same set as Stack Overflow)

### tldr-pages
- Passthrough ‚ÄĒ already clean, curated content.

## Tokenizer

Custom BPE, 32,016 tokens trained on this corpus:
- 32,000 BPE merges
- 16 special tokens: `<|bos|>` (0), `<|eos|>` (1), agent protocol tokens (2‚Äď13), `<|done|>` (14), `<|pad|>` (15)

## Format

```python
import numpy as np

train = np.fromfile("corpus_train.bin", dtype=np.uint16).reshape(-1, 6144)  # (173911, 6144)
val   = np.fromfile("corpus_val.bin",   dtype=np.uint16).reshape(-1, 6144)  # (19340, 6144)
```

- `dtype`: uint16 (2 bytes/token; vocab fits in 16 bits)
- `shape`: `[N, 6144]` ‚ÄĒ each row is a packed sequence of complete documents
- Documents separated by `<|eos|>` (ID=1); ~6 docs/chunk on average
- No padding ‚ÄĒ 100% token utilisation

## Train/Val Split

- 90/10 split at **document boundaries** ‚ÄĒ no document straddles the split
- Shuffled at document level with `seed=42` before splitting
- Ensures val is representative across all sources

## Attention Masking

Documents are packed contiguously. At training time a block-diagonal causal mask prevents attention across document boundaries:

```python
is_eos = (x == EOS_ID)
doc_id = torch.cat([zeros, is_eos[:, :-1].cumsum(dim=1)], dim=1)  # (B, T)
mask = (doc_id.unsqueeze(2) == doc_id.unsqueeze(1)) & causal       # (B, 1, T, T)
```

Note: custom mask disables FlashAttention in SDPA. For T > ~1024 (3b/7b configs), switch to `flash_attn_varlen_func`.
